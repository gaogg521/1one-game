# AI 接力索引（先搜「前人做过什么」——思路模板）

**定位**：这是一套 **检索习惯 + 索引模板**，便于后来的助手 **多渠道**找回上下文。**不限定**记忆必须放在 Git 仓库或本目录——你可把同一张「证据分层表」复制到 Wiki、Notion、本地笔记路径；核心是 **别只依赖聊天记录**。

**若记忆恰好落在本仓库的 `PROJECT_MEMORY/`**，下文路径可直接用；若在别处，把表格里的路径改成你的约定即可。

---

## 一、证据分层（从高到低，尽量都看）

| 优先级 | 去哪找（示例路径） | 通常能发现什么 |
|--------|-------------------|----------------|
| P0 | **`NEXT_ACTION.md`**（或与用户约定的「下一步」文件） | 上一棒指定的第一件事、阻塞项 |
| P0 | **`CURRENT_STATUS.md`** | 整体进度、构建/测试、已知问题 |
| P1 | **`iterations/`** | 按日期归档的迭代叙事（若有） |
| P1 | **`SESSION_LOG.md`** | 近期操作流水 |
| P1 | **`TASK_QUEUE.md`** | 待办优先级 |
| P2 | **`DECISIONS.md`** | 架构与技术结论 |
| P2 | **`docs/`** | 如 `docs/ai-handoff-architecture-cn.md`、`docs/recent-progress.md` |
| P2 | **仓库根 `CLAUDE.md`、`AGENTS.md`** | 流程与框架约束 |
| P3 | **版本历史** | `git log`（若使用 Git） |
| P3 | **代码内痕迹** | `TODO`、`FIXME`、`NOTE` 等（按需） |

## 二、开工检索清单（可复制到任意工具）

动手写功能前，尽量完成（可并行）：

1. [ ] 读 **下一步 / 状态**（约定路径下的 `NEXT_ACTION` 或等价物）。  
2. [ ] 浏览 **最近迭代归档**（若有）。  
3. [ ] 看 **`SESSION_LOG` 末尾**（若有）。  
4. [ ] 扫 **`TASK_QUEUE`**（若有）。  
5. [ ] 架构相关则读 **决策记录**（若有）。  
6. [ ] 仍不清晰：**`docs/`** + **`git log`**（若有）+ **`TODO` 搜索**。  
7. [ ] （可选）在 **`SESSION_LOG`** 留一句「已读过哪些来源」，方便下一棒。

## 四、若几乎没有结构化记忆怎么办？

1. **`docs/`**、项目 **`README`**。  
2. **`git log` / diff**（若有 Git）。  
3. **全局 `TODO` / `FIXME`**。  
4. 若之后采用 **`PROJECT_MEMORY/`**，可把结论 **回填** 到 `CURRENT_STATUS` / `NEXT_ACTION`，方便后来者。
