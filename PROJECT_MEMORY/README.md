# PROJECT_MEMORY 说明

本目录是 **在本仓库内放置 AI 协作记忆的「可选」方案**：进度、队列、决策与下一步可以写在这里，也可以写在团队约定的其它位置。

## 若使用本目录：建议先看 `INDEX.md`

[`INDEX.md`](INDEX.md) 里是 **多源检索清单**（不绑定 Git）：后来的助手应先 **搜索 / 阅读** 多渠道证据，而不假设「只有一个文件能说清楚」。

## 跨工具接力（原则）

- **习惯**：开工前尽量读完 **约定路径下的状态 + 文档 +（若有）版本历史 + TODO**。  
- **形式**：是否在 Git 里维护、是否 push，由项目决定。  
- **安全**：勿在此写入密钥与隐私原文。

## 建议阅读顺序（当记忆在本目录时）

1. [`INDEX.md`](INDEX.md)  
2. `NEXT_ACTION.md` → `CURRENT_STATUS.md` → `TASK_QUEUE.md` → `iterations/` → `DECISIONS.md` → `SESSION_LOG.md`  

完整 CTO 流程见仓库根 **`CLAUDE.md`**。

## 维护约定

按项目节奏更新；里程碑可写 **`iterations/`**；结束前可选用 **[`SESSION_END_CHECKLIST.md`](SESSION_END_CHECKLIST.md)**。
